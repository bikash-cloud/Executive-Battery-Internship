Fault-Tolerant Embedded Runtime System

An ESP32-based runtime subsystem that detects hardware and software failures, isolates faulty modules, maintains timestamped fault logs, and transitions between NORMAL, DEGRADED, FAILSAFE, and SHUTDOWN modes.

Open Wokwi Simulation View Source Code

Project Overview

This project implements a fault-tolerant embedded runtime system capable of detecting sensor disconnection, invalid ADC readings, frozen ADC conditions, and relay mismatch faults. The controller autonomously isolates faulty modules while allowing healthy modules to continue operating.

Fault Detection Engine

Sensor Disconnect

Detects ADC values near 0V or full-scale voltage.

Invalid Reading

Detects readings outside valid operating voltage range.

Frozen ADC

Detects ADC values stuck for a long period.

Relay Mismatch

Compares relay command and feedback state.
