Project Overview

The Event-Driven Safety Protection Kernel is an embedded safety controller built on ESP32 for automotive battery protection. The controller operates entirely without delay() by using a millis()-based scheduler. It continuously monitors four battery cells and detects abnormal operating conditions including weak cells, over-voltage, sensor failures, and rapid voltage spikes. Upon fault detection it performs safe relay isolation, activates audible alarms, displays warning messages on the LCD, and automatically restores normal operation after safe recovery while preventing relay chatter.

The firmware is structured around a safety state machine and independent millis()-based software tasks. ADC monitoring, LCD updates, buzzer patterns, relay control, and recovery timing run without blocking the main loop.

Event Driven Timing

ADC Task

Reads all four battery cell voltages at fixed intervals using millis().

LCD Task

Updates system warnings and safety status without using delay().

Buzzer Task

Generates fault and recovery alerts using non-blocking timing.

Recovery Task

Waits for stable voltage conditions before allowing relay reconnection.

The system remains in NORMAL mode during healthy operation. When any unsafe event is detected, the controller enters FAULT mode, disconnects the relay, activates the buzzer, and displays the fault message. After the fault clears, the controller enters RECOVERY mode and waits for a fixed safe interval before reconnecting the relay.
